## Queries * Request information from a single service * Usually synchronous Note: Query's van de UI naar de backend hier los van zien, de UI is onderdeel van het component.
## Commands * Request for something to happen * Are sent to a single destination * May return information about the result
## Events * Describe something that has happened * For everybody that is interested * Publisher doesn't care if somebody listens
## Building reactive systems

### Start thinking about events * Explore what happens in the system * Explore what triggers events * Group events * Find out how events flow trough the system
